## 完成版までの残作業(優先度順)

### P1 — 品質:既知のレイアウト解析(M1)不具合
1. ~~**Table キャプションの行混在**(最重要)~~ **解決済み(2026-07)**
   行クラスタリングを「移動平均中心±許容」から「垂直スパンの実重なり
   (文字高と行内最大文字高の小さい方の50%以上)」に変更(`_cluster_rows`)。
   上付き文字は本文行に残しつつ、隣接する異サイズ行のドリフト連鎖を遮断。
   併せて単語間スペース判定を行内の文字幅中央値でスケールさせ(8pt行の
   スペース欠落を解消)、表ヘッダの heading 誤分類を「Table キャプション直下は
   data」ルールの拡張で修正。クロスページ結合が表キャプションに分断される
   問題も M1 の継続フラグを使う後処理で解消。**残存英語は 48語 → 1語**。
2. ~~**上付き引用番号による重複ブロック**~~ **解決済み(2026-07)** — 1. と同じ
   クラスタリング変更で消滅(p.4 の重複ブロックが正常な1ブロックに)。
3. **タイトルページの散在断片**(著者行の a/b/c/d、段落端の引用番号)
   CLAUDE.md 記載の既知課題。フラグメント除去(pass 2)の「両隣が削除済み」条件を
   「近傍 n 個中の多数が削除済み」に緩めるのが有力。
4. **単語分割の残骸**("Spratf / ord" のような行またぎ分割)
   英語除去後に残ったオペを再結合しないため。除去対象ユニットに部分一致する
   短いオペも kill 対象に含める(現在は3文字以下のみ)。
5. **表キャプション訳文と表罫線の接触**(軽微)
   日本語化で行数が増えたキャプションが表の上罫線にわずかに掛かる(paper p.2)。
   罫線(ベクタ線)は figure として検出されないため衝突回避が効かない。
   → 罫線・矩形も障害物として region_box に渡す。

### P2 — 機能:実用に必要

   レイアウト(テキストボックス散在、背景色、縦横比)への placement 対応。
6. ~~**翻訳キャッシュ**~~ **解決済み(2026-07)** — content-hash キャッシュ
