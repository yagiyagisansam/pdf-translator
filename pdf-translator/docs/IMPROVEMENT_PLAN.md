# 修正案 — 完成版に向けた改善計画

## 2026-07 再構築: 4ロール・アーキテクチャ + 実論文対応

参照ゴール(別システムの日英変換PDF)を1つの目標に設定し、役割分担型で再構築した。

**4ロール(`src/roles/`)** — 各ロールは単一の入口を持つモジュール、`orchestrator` が連結:
- **翻訳者 `translator_role`**: 英語→日本語(⟦Tn⟧保護・復元、参考文献は非翻訳)。無料Google既定、キーがあればClaude。
- **PDF製作者 `producer`**: PDF→レイアウト仕様(段組レーン・図表/保持テキストの障害物ボックス・読み順)。図表座標は原文のまま。
- **編集者 `editor`**: 図表を原位置に保ち、日本語を衝突なく配置(実証済みm3配置エンジンに委譲)。実描画行を `_placed.json` に出力。
- **確認者 `qa`**: 品質を保証し責任を持つ。①日本語妥当性(未翻訳ゼロ・⟦Tn⟧残存ゼロ・描画到達・残存英語≤閾値、任意でLLMの日本語自然性チェック)②配置忠実性(図保持・行/行&行/図の重なりゼロ・オーバーフローなし)を検査。**英語との意味照合はしない方針**。不適合時は各ロールに修正を要求し、フォント下限を下げる/再翻訳するなどして**最大N回自動リトライ**。
- **Web**: `webapp` は orchestrator をサブプロセス実行し、4ロールの進捗を日本語表示。

**実論文対応の本質的修正(サブセットフォント)**: pikepdf が返す生バイトはフォント固有コードで、pdfplumber の Unicode と一致しないため、内容ベースの英語除去が実論文で効かなかった。各フォントの **ToUnicode CMap でオペを Unicode 復号**してから照合するよう修正(`m3_generate._page_font_decoders` / `_decode_op`)。実ED論文(12頁)で残存英語 **147語→23語**、1頁目の要旨の英語透けが解消。合字グリフ ﬁ/ﬂ を引用符転用するフォント向けに展開版/除去版の二重正規化も追加。

**解決済み(この再構築で判明→修正)**:
- ~~一部ブロックで pdfplumber の文字抽出順序が崩れる~~ **修正済み**: 原因は `_cluster_rows` が行のバウンディングボックスを下方向に拡張し続け、行間の狭い箇所で連続する複数行(実ED論文で230文字=約4行)を1行に連鎖併合し、`_mkline` の x0 ソートで文字が交互化していたこと。行の基準を累積箱から**行中心の中央値(行内で安定)**に変更して連鎖を遮断。実ED論文2頁目の緒言が正常化し残存英語が解消。
- ~~表がある頁で左右段が重なる~~ **修正済み**: 左右レーンのx範囲が(ガターを越えたブロックにより)重なる場合、`producer.lanes_for_page` でガター中点にクランプ。実ED論文6頁目の段組衝突を解消。
- **結果**: 実ED論文(12頁・サブセットフォント)が確認者を**1ラウンドで通過**(残存英語ゼロ・重なりゼロ・図表保持)。サンプルはM4 5/5維持。
- ~~参照ゴールは構造保持リフロー~~ **本格リフロー実装済み(2026-07)**: `editor` を段組ゾーニングを解いた列フローに刷新。図表は原座標固定のまま、日本語を読み順で全幅帯(タイトル/著者/所属/要旨)と二段帯(本文)に流し込み、障害物(図・保持テキスト)のyバンドを回避。フロア文字サイズでも入らない場合は従来の領域配置にフォールバックし品質後退を防止。paper/ED両方で図保持+二段リフローを確認(参照ゴールに近い)。M4回帰なし(5/5)。

   行クラスタリングを「移動平均中心±許容」から「垂直スパンの実重なり
   (文字高と行内最大文字高の小さい方の50%以上)」に変更(`_cluster_rows`)。
   上付き文字は本文行に残しつつ、隣接する異サイズ行のドリフト連鎖を遮断。

   スペース欠落を解消)、表ヘッダの heading 誤分類を「Table キャプション直下は
   data」ルールの拡張で修正。クロスページ結合が表キャプションに分断される
   問題も M1 の継続フラグを使う後処理で解消。**残存英語は 48語 → 1語**。
2. ~~**上付き引用番号による重複ブロック**~~ **解決済み(2026-07)** — 1. と同じ
   クラスタリング変更で消滅(p.4 の重複ブロックが正常な1ブロックに)。

   CLAUDE.md 記載の既知課題。フラグメント除去(pass 2)の「両隣が削除済み」条件を
