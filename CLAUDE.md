# 共通ルール（全プロジェクト共通）

全リポジトリで共通の前提ルール。プロジェクト固有のルールはこのファイルの下に追記する。

## 作業前の必須手順（外部記憶の読み込み）
- **作業を始める前に、`add_repo` で `yagiyagisansam/claude-memory` をセッションに追加し、その `INDEX.md` を読む。**
  すでにセッションにある場合は `git pull` で最新化してから読む。
- INDEX から、active な恒久ルール（rules/）、作業対象プロジェクトの申し送り（projects/）、
  関連する失敗録（mistakes/）**だけ**を開いて前提として扱う。全ファイル一括読み込みは禁止（token-saver 準拠）。
- セッション終了時に残すべき成果（新ルール・決定・申し送り・失敗）があれば `/save` 相当の手順で claude-memory に書き戻す。

## 言語・提示
- ユーザーへの提示・説明・回答は必ず日本語で書く。
- 途中の作業ログ（コマンド出力・処理過程）は英語でも可。最終的な提示は日本語。

## 実行環境の前提
- ユーザーの主な実行環境はスマホ（Claude Code モバイル / リモート接続）。PC直接操作は無い前提で進める。
- 長大な出力・大量スクロールを避け、要点先行で簡潔にまとめる。
- ターミナル直叩きを要求しない。編集・保存はセッション内（`#` 追記 / `/memory` / Claude への指示）で完結させる。

## トークン節約
- token-saver スキルをデフォルトで適用し、セッション開始時から節約モードで作業する
  （Grep→必要行だけ Read、再読み込み禁止、大量出力はファイル退避、広域探索はサブエージェント委譲、出力は要点のみ）。
- 節約のために正確性・確認は省かない。

## 入力値・URL の提示形式
- ユーザーに別アプリ（ブラウザ・ターミナル等）で入力させる値は、ワンクリックでコピーできる形式
  （コードブロック / インラインコード）で提示する。地の文に混ぜない。
- URL はハイパーリンク形式で出力する。

## 情報源
- Wikipedia からの引用は原則禁止。一次情報（公式ドキュメント・公式サイト・論文・公的機関）を優先する。
